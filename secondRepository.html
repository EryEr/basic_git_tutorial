Creating another branch in a repository

Creating another branch as an example for collaboration working style via checkout -b secondBranch command



After adding all the necessary update and changes in this branch we need to create a PR (Pull request, PS:in some company they may call as MR for merge request words)





Your colleagues who appointed as a reviewer may put some commands, or they may mention issues for your updates like below









After changes for the improvement of these issues needs to push all changes into the branch again and asking review again.

After colleagues acceptance, merge button turn to green. Then you can merge all the changes from secondBranch to the master branch

After mege process we can delete the secondBranch with git -d secondBranch command
